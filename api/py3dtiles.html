py3dtiles.convert.convert(files, outfolder='./3dtiles', overwrite=False, jobs=4, cache_size=1544, srs_out=None, srs_in=None, fraction=100, benchmark=None, rgb=True, graph=False, color_scale=None, verbose=False)[source]¶

Convert pointclouds (xyz or las) to 3dtiles tileset containing pnts node

Parameters

  • files (list of str, or str) – Filenames to process. The file must use the .las or .xyz format.

  • outfolder (path-like object) – The folder where the resulting tileset will be written.

  • overwrite (bool) – Overwrite the ouput folder if it already exists.

  • jobs (int) – The number of parallel jobs to start. Default to the number of cpu.

  • cache_size (int) – Cache size in MB. Default to available memory / 10.

  • srs_out (int or str) – SRS to convert the output with (numeric part of the EPSG code)

  • srs_in (int or str) – Override input SRS (numeric part of the EPSG code)

  • fraction (int) – Percentage of the pointcloud to process, between 0 and 100.

  • benchmark (str) – Print summary at the end of the process

  • rgb (bool) – Export rgb attributes.

  • graph (bool) – Produce debug graphes (requires pygal).

  • color_scale (float) – Force color scale

Raises

SrsInMissingException – if py3dtiles couldn’t find srs informations in input files and srs_in is not specified
